# UX Study Lifecycle (Intended)

This document describes the intended end-to-end flow for a GazeOps study.

1. **Researcher creates study**
   - Defines objective, study name, and target page(s).
2. **Researcher defines page/task/AOIs**
   - Adds task prompts and marks areas of interest relevant to success criteria.
3. **Tester consents**
   - Tester is informed about telemetry capture and privacy boundaries.
4. **Tester calibrates**
   - Calibration points are collected to estimate baseline tracking quality.
5. **Tester performs task**
   - Tester completes the assigned UX task on the target page.
6. **Telemetry is recorded**
   - Gaze and interaction events are captured as structured telemetry.
7. **Quality score is computed**
   - Session quality metrics summarize confidence and data reliability.
8. **Report is generated**
   - System assembles session-level summary and AOI/task metrics.
9. **Researcher reviews results**
   - Researcher evaluates outcomes, quality caveats, and next design actions.
