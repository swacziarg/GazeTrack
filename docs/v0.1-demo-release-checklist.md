# `v0.1-demo` Release Checklist

## Release framing

- [x] README leads with synthetic telemetry demo pipeline language.
- [x] Product spec separates current demo, experimental browser gaze spike, and future product requirements.
- [x] Roadmap reflects completed baseline plus next milestones instead of an old speculative 8-week plan.
- [x] Backend and frontend READMEs label analytics/replay as demo-grade and privacy-safe.
- [x] Interview story describes a synthetic telemetry demo pipeline, not live production webcam tracking.

## Privacy and safety

- [x] Synthetic mode does not require webcam permission.
- [x] Browser gaze experiment is feature-flagged, consent-gated, guarded against missing `window.webgazer`, and not bundled.
- [x] Docs state that raw webcam video, frames, images, screenshots, blobs, and base64 media must not be persisted or returned.
- [x] Backend rejects media-like payload keys before persistence.
- [x] Docs avoid medical-grade, biometric identity, perfect-accuracy, and production webcam tracking claims.

## Implemented demo surface

- [x] React/Vite/TypeScript demo dashboard.
- [x] Synthetic calibration/session flow and quality modes.
- [x] FastAPI + SQLite persistence for current demo entities and report payloads.
- [x] Accepted telemetry ingestion and persisted backend report generation.
- [x] Current report fields include event counts, AOI hit metrics, bounded raw dwell, demo fixations, TTFF from task start, quality verdicts, privacy summary, and schematic replay payloads.
- [x] Replay is documented as normalized-coordinate schematic playback only.

## Explicitly not claimed for `v0.1-demo`

- [x] Production-grade webcam tracking.
- [x] Medical-grade eye tracking.
- [x] Real heatmaps.
- [x] Screenshot/video replay.
- [x] DOM-derived AOI detection.
- [x] CAF delay.
- [x] Authentication/authorization.
- [x] Deployment.
- [x] Export/share features.

## Validation status

- [x] Backend unit/integration tests pass.
- [x] Frontend tests pass.
- [x] Frontend production build passes.
- [x] E2E happy path with backend running.
- [x] Backend report response validation against `contracts/session-report.schema.json`.

Exact validation commands:

```bash
cd backend
PYTHONPATH=. pytest
PYTHONPATH=. pytest tests/test_session_report_schema_contract.py
```

```bash
cd frontend
npm run test
npm run build
npm run e2e
```

The Playwright E2E starts FastAPI on `127.0.0.1:8000` with
`GAZETRACK_DATABASE_URL=sqlite:////private/tmp/gazetrack-e2e.db` using the documented `backend/.venv`, starts Vite on
`127.0.0.1:5173`, keeps the tracker in default synthetic mode, completes the synthetic session, and checks that
ingest/report UI reaches the persisted backend success state. It does not enable `VITE_ENABLE_WEBGAZER`.
